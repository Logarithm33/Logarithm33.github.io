P3 用Logisim搭建单周期cpu

设计要求

处理器为32位单周期处理器，支持add,sub,ori,lw,sw,beq,lui,nop指令，其中nop为空指令，add,sub为无符号运算
根据要求，我应该设计以下几种模块

模块设计

IFU（取指单元）
内部包含PC（程序计数器）用寄存器实现，地址范围0x00003000-0x00006FFFF
IM用ROM实现，容量为4096*32bit,地址从0开始

这里将PC中储存的地址转换为IM的地址，采用的是先PC-0x00003000，由于每条指令占四个字节所以右移2位（除以4）

信号名 方向 位宽 说明
reset I 1 异步复位信号
clk I 1 时钟信号
nextpc I 32 下一条指令所在地址
pc O 32 目前指令所在地址
Instr O 32 目前指令信号

GRF（通用寄存器堆）
要求同P0课下完成的GRF,直接copy

ALU（算数逻辑运算单元）
要在该模块中实现32位加、减、或、与、比较大小的功能

信号名 方向 位宽 说明
A I 32 运算数A
B I 32 运算数B
ALUop I 4 操作码
result O 32 结果
zero O 1 如果A=B，置1
运算编码 运算操作
0000 A + B
0001 A - B
0010 A
0011 A & B
0100 立刻将B传入的16位数传入A的高位

DM（数据储存器）
使用RAM实现，容量为3072 × 32bit，应具有异步复位功能，地址采取先右移两位字对齐，然后截断低12位的方式寻址

信号名 方向 位宽 说明
WriteAddress I 32 在RAM中写入数据的地址
WriteData I 32 写入的数据
MemWE I 1 是否储存
clk I 1 时钟信号
reset I 1 异步复位信号
ReadData O 32 读出的数据

EXT（扩展单元）

信号名 方向 位宽 说明
in I 16 16位原数据
EXTop I 1 为1则符号扩展，为0则零扩展
out O 32 扩展后数据

NEXTPC（寻址单元）
负责计算下一条指令的地址，目前分为两种情况，一种正常顺序执行PC = PC + 4
另一种遇到beq指令 PC = PC + 4 + (immediate16 << 2)

信号名 方向 位宽 说明
PC I 32 当前指令地址
immediate16 I 16 beq指令相对偏移量
OP I 1 为1则执行beq跳转，为0执行顺序跳转
nextPC O 32 下一条指令地址

Controller(控制信号)
由于nop为空指令，因此与控制信号无关
负责根据每条指令的OPcode生成对应的控制信号，将解码逻辑分为和逻辑和与逻辑
和逻辑负责根据OPcode译码为指定的指令信号

OPcode 指令
000000 R型指令
001101 ori
001111 lui
100011 lw
101011 sw
000100 beq

对r型指令，根据func译码为指定指令信号，并与OPcode输出信号用与门连接

func 指令
100000 add
100010 sub

与逻辑负责根据不同的指令信号生成控制信号

控制信号 说明
RegDst 为1使用寄存器rd，为0使用寄存器rt
RegWE GRF写入使能信号
ALUsrc 为1使用imme16经EXT扩展结果，为0使用寄存器读出数据ReadData
ALUop 根据opcode和func生成
DmWE DM写入使能信号
MemToReg 为1将内存读出数据写入寄存器堆
Branch 为1代表beq指令，为0代表其他指令
指令 RegDst RegWE ALUsrc ALUop DmWE MemToReg Branch
add 1 1 0 0000 0 0 0
sub 1 1 0 0001 0 0 0
ori 0 1 1 0010 0 0 0
lui 0 1 1 0100 0 0 0
lw 0 1 1 0000 0 1 0
sw x 0 1 0000 1 x 0
beq x 0 0 x 0 x 1

P3课上

P3课上做了二周目，Logisim确实对我来说不好用
课上三题基本分为 只需要修改ALU和ctrl的计算类指令，修改npc的条件跳转指令（有可能含有条件写入），对dm读出数据进行处理的访存指令
计算类指令没什么好说的，按照题目条件来就行了
条件跳转指令也比较公式，ctrl连出一条新信号和cmp的新信号用与门连接和npc连起来即可，我掉的小坑在条件写入如果不写入的话，grf的使能一定要是0，不然评测机会输出错误的写入信息
访存指令可能会涉及for循环，Logisim写for循环有点烦，我是看循环数不高直接枚举了，侥幸通过，从此不碰Logisim

课上碎碎念

第一次闯关上机就寄了，说实话心里确实没底，主要很快做出来一道之后在第二题直接卡到最后，下一次上机心里也没底能过，有点烦
卡关之后回来也对课下cpu做了很多优化，主要就是留出了ctrl中给新信号的预留位置，这样课上加指令不会在这些重复的部分浪费时间，这种预留思想也一直延续到了所有之后的上机，确实省了很多麻烦
后续的上机也证明了，P3卡关确实不是什么坏事，甚至能说因祸得福，给每次课下都留出了两周的充裕时间和中测保证课下正确性，反而对之后每次课上的上机更有信心了
总之核心思想就是，不挂P一定是好事，但挂P不一定是坏事，放宽心
