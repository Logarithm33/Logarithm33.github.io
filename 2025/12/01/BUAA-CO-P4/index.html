P4 用 Verilog 搭建单周期 CPU

P4 基本就是把 P3 翻译到 Verilog，按模块进行翻译即可。

我个人觉得不用特别注意命名规范，到 P5 基本没有能复用的模块。

设计要求

处理器为 32 位单周期处理器，支持 add, sub, ori, lw, sw, beq, lui, jal, jr, nop 指令。要求和 P3 课下基本相同，多加了 jal 和 jr 两条指令。

这里只给出新加或修改的模块，其他参照 P3 的博客。

模块设计

IFU（取指单元）

splitter（分位器）

GRF（通用寄存器堆）

ALU（算术逻辑运算单元）

DM（数据储存器）

EXT（扩展单元）

NPC（寻址单元） 负责计算下一条指令的地址，在加入 jal 和 jr 后分为四种情况：

指令 PC 计算方式
add,sub,ori,lw,sw,lui,beq(A!=B) PC = PC + 4
beq(A==B) PC = PC + 4 + (immediate16 << 2)
jal PC = { (PC + 4)[31:28], immediate26, 2'b00 }
jr PC = $31

Controller(控制信号) 由于 nop 为空指令，因此与控制信号无关。

负责根据每条指令的 OPcode 生成对应的控制信号，将解码逻辑分为和逻辑和与逻辑。

和逻辑（指令译码）

负责根据 OPcode 译码为指定的指令信号。

OPcode 指令
000000 R 型指令
001101 ori
001111 lui
100011 lw
101011 sw
000100 beq
000011 jal

对 R 型指令，根据 func 译码为指定指令信号，并与 OPcode 输出信号用与门连接。

func 指令
100000 add
100010 sub
001000 jr

与逻辑（控制信号生成）

负责根据不同的指令信号生成控制信号,这里我把jal的PC+4也放到ALU里计算了，实际上更推荐单独走一条通路，简化控制信号设计

控制信号 位数 说明
RegDst 2 2'b00 : rt
2'b01 : rd
2'b10 : $31
RegWE 1 GRF 写入使能信号
ALUsrcA 2 2'b00 : GRF[rs]
2'b01 : pc值本身
2'b10 : 0
2'b11 : 0
ALUsrcB 2 2'b00 : GRF[rt]
2'b01 : 经过EXT的32位立即数
2'b10 : 4
2'b11 : 0
ALUop 4 在ALU和ctrl中根据opcode和func生成
DmWE 1 DM 写入使能信号
MemToReg 1 1'b1 : DM[ReadAddress]
1'b0 : ALUresult
Branch 1 beq 跳转信号
Jump 1 Jal 跳转信号
EXTop 1 1'b1 : 符号扩展
1'b0 : 零扩展
Jrsign 1 Jr 信号

ALU 运算编码

ALUop 操作
4'b0000 A + B
4'b0001 A - B
4'b0010 A | B
4'b0011 A & B
4'b0100 { B[15:0], 16'h0000 }

指令控制信号表

指令 RegDst RegWE ALUsrcA ALUsrcB ALUop DmWE MemToReg Branch Jump EXTop Jrsign
add 01 1 00 00 0000 0 0 0 0 0 0
sub 01 1 00 00 0001 0 0 0 0 0 0
ori 00 1 00 01 0010 0 0 0 0 0 0
lui 00 1 00 01 0100 0 0 0 0 0 0
lw 00 1 00 01 0000 0 1 0 0 1 0
sw 00 0 00 01 0000 1 0 0 0 1 0
beq 00 0 00 00 0001 0 0 1 0 1 0
jal 10 1 01 10 0000 0 0 0 1 0 0
jr 01 0 00 00 0000 0 0 0 0 0 1
nop 00 0 00 00 0000 0 0 0 0 0 0

一直到P4我都采用的是以指令为主的译码方式，即对于每一条指令，给出对于该指令的所有控制信号取值

P4课上

太久远了没啥印象了，反正指令形式和P3基本一样，换成verilog描述之后老老实实把RTL式子翻译成代码即可，比logisim舒服太多
P4还是单周期cpu，加新指令完全无需考虑前后指令的影响，只需要处理好每一条指令的数据通路即可
